# Mnemosyne v6.1 Release Notes

> Version: 6.1.0
> Date: 2026-05-08
> Theme: Predictive memory, multi-dimensional retrieval, and faster dream consolidation

Mnemosyne v6.1 is a performance and quality release on top of v6.0. It keeps the v5.0 foundation — L0/L1/L2 layered memory, MCP integration, REST API, dashboard, and dream logs — then upgrades the memory core from "store and retrieve" to "predict, verify, and evolve".

## What Changed Since v5.0

v5.0 made Mnemosyne usable as an AI memory system. v6.0 and v6.1 make it behave more like a living memory system.

| Area | v5.0 | v6.1 |
|------|------|------|
| Memory model | Layered context + knowledge graph | Predictive memories with confidence, verification, and contradiction handling |
| Search | Vector, keyword, hybrid | Hybrid plus precise/creative spreading activation across graph dimensions |
| MCP tools | 4 tools | 6 tools, adding update and delete |
| Dream | Batch consolidation | Fast/Slow dual-stream consolidation |
| Vector routing | Python/numpy scans | FAISS-backed VectorIndex with numpy fallback |
| Graph metadata | Relation type only | Relation type + graph dimension + connection strength |

## Highlights

### Predictive Memory

A memory can now describe when it applies and what it predicts:

```python
memory_write(
    content="torch 2.11.0 crashes on this Windows setup; use torch 2.6.0 instead",
    precondition="installing torch on Windows",
    predicted_outcome="torch 2.6.0 is the stable choice"
)
```

When a later memory matches the same precondition, Mnemosyne checks whether the old prediction still holds. Confirmation increases confidence. Conflict creates a `contradicts` edge and lowers stale confidence.

### Five Search Modes

`memory_search` now supports:

| Mode | Behavior |
|------|----------|
| `hybrid` | Default vector + keyword + graph retrieval |
| `precise` | Conservative spreading through strong graph edges |
| `creative` | Wider association through weak edges and `is_a` concept jumps |
| `vector` | Pure semantic similarity |
| `keyword` | FTS5 keyword search |

Search can also filter by graph dimension:

```python
memory_search(
    query="API gzip parse failure",
    mode="precise",
    graph_dim="causal",
    layer="L0"
)
```

### Six MCP Tools

The MCP server now exposes:

| Tool | Purpose |
|------|---------|
| `memory_write` | Store experiences, corrections, raw observations, and predictive memories |
| `memory_search` | Retrieve memories with layer, mode, graph dimension, and tag filters |
| `memory_inject` | Load relevant memory chains at session start |
| `memory_detail` | Expand selected memories to full L2 content |
| `memory_update` | Update content, confidence, principle, or context tags |
| `memory_delete` | Delete a node and its edges |

### Fast/Slow Dream Pipeline

Dream consolidation is now split into two streams:

| Stream | Purpose |
|--------|---------|
| Fast Path | Deterministic maintenance: decay, sync, incremental association, safe index cleanup |
| Slow Path | Deeper consolidation: contradictions, causal links, strategy distillation, optional LLM review |

This follows a three-layer biomimetic design:

| Brain analogy | Mnemosyne implementation |
|---------------|-------------------------|
| Hippocampus | Fast write-time validation and auto-association |
| REM sleep | Incremental replay of new memories against the graph |
| Prefrontal cortex | Optional LLM-assisted semantic judgment |

### Faster Vector Routing

v6.1 introduces `VectorIndex`, backed by FAISS when available and numpy when not. It is used for:

| Path | Benefit |
|------|---------|
| Add-node merge detection | Avoids full-table vector scans |
| Predictive validation | Finds precondition matches quickly |
| Dream association phases | Processes new memories incrementally |
| Search | Faster candidate retrieval for graph spreading |

## Upgrade From v5.0

1. Back up your database.
2. Run the migration script:

```bash
python scripts/migrate_v50_to_v60.py
```

3. Verify the graph:

```bash
python scripts/graph_dream.py --stats
```

4. Optional: install FAISS acceleration:

```bash
pip install faiss-cpu
```

If FAISS is not available, Mnemosyne automatically falls back to numpy search.

## MCP Configuration

```json
{
  "mcpServers": {
    "mnemosyne": {
      "command": "python",
      "args": ["scripts/mcp_server/start_mcp.py"]
    }
  }
}
```

For absolute-path configs, point `args` to your local `scripts/mcp_server/start_mcp.py`.

## Notes For Existing Users

- Existing v5.0 memories remain compatible after migration.
- New predictive fields are optional; normal `memory_write(content="...")` still works.
- `memory_search` defaults to `hybrid`, so old search usage continues to work.
- `memory_update` and `memory_delete` are powerful tools; expose them only to trusted agents.
- LLM review is optional. The core graph, search, and dream pipeline can run locally without external APIs.
