# Mnemosyne v6.0 Architecture Design

> Version: 6.0.0 | Date: 2026-05-07 | Status: Draft

## 1. Overview

Mnemosyne v6.0 represents a major architectural evolution from v5.0, incorporating four cutting-edge research directions validated in 2025-2026:

| Direction | Source | What It Enables |
|-----------|--------|-----------------|
| **MAGMA** Multi-Graph Orthogonal | ACL 2026 | Separate semantic/temporal/causal/entity graphs for precise retrieval |
| **A-MEM** Memory Evolution | NeurIPS 2025 | Old memories update when new related memories arrive (not append-only) |
| **SmartVector** Metadata | Microsoft 2026 | Confidence decay, half-life, verification tracking per memory |
| **SYNAPSE** Spreading Activation | Jan 2026 | Brain-like activation spreading for deeper associative retrieval |

**Core philosophy shift**: From "store and retrieve" to "predict, verify, and evolve."

## 3. Data Model Changes

### 3.1 nodes Table — 8 New Fields

| Field | Type | Default | Purpose | Source |
|-------|------|---------|---------|--------|
| `confidence` | REAL | 1.0 | How reliable this memory is (0-1). Decreases on contradiction, increases on verification | SmartVector |
| `verified_at` | TEXT | NULL | Last time this memory was validated against new experience | SmartVector `last_validated` |
| `verified_count` | INTEGER | 0 | How many times new experience confirmed this memory | SmartVector `positive_feedback` |
| `half_life_days` | REAL | 30.0 | Ebbinghaus half-life in days. Higher = forgets slower | SmartVector |
| `precondition` | TEXT | NULL | Environmental condition under which this memory applies. Enables predictive coding | Predictive Coding |
| `predicted_outcome` | TEXT | NULL | What we predict will happen when precondition is met | Predictive Coding |
| `context_tags` | TEXT | '[]' | JSON array of tags for filtered retrieval (e.g. ["python","windows"]) | A-MEM Note Construction |
| `precondition_vec` | BLOB | NULL | Pre-encoded 1024-dim float32 vector of precondition text | MAGMA |

#### Field Interactions

```
precondition + predicted_outcome + precondition_vec = "Predictive Memory"
  "When installing torch on Windows (precondition),
   I predict torch 2.6.0 is the only working version (predicted_outcome)"
  precondition_vec = embed("installing torch on Windows")

confidence + verified_at + verified_count + half_life_days = "Smart Decay"
  A memory verified 10 times over 3 months is far more reliable
  than one accessed 50 times but never verified
```

### 3.2 edges Table — 2 New Fields

| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `graph_dim` | TEXT | 'semantic' | Which orthogonal graph this edge belongs to: semantic/temporal/causal/entity |
| `strength` | TEXT | 'strong' | Connection strength: strong (reliable) or weak (exploratory) |

### 3.3 Dimension Mapping

```
relation_type    → graph_dim    strength
─────────────────────────────────────────
similar_to       → semantic     strong (weight >= 0.6) / weak (< 0.6)
is_a             → semantic     strong
caused           → causal       strong
solves           → causal       strong
contradicts      → causal       strong
transfers_to     → entity       weak
evolved_from     → semantic     weak
temporal (new)   → temporal     strong
```

## 4. Core Interface Changes

### 4.1 AbstractGraphStore — New/Changed Methods

```python
class AbstractGraphStore(ABC):
    # Existing methods with signature changes
    def add_node(self, content, node_type="experience",
                 task_type=None, project=None, tags=None, principle=None,
                 # NEW parameters:
                 precondition=None,
                 predicted_outcome=None,
                 context_tags=None,
                 **kwargs) -> str:
        """Write node. New: auto-encodes precondition_vec,
        checks predictive validation against existing memories."""

    # NEW methods
    @abstractmethod
    def update_node(self, node_id: str, **fields) -> bool:
        """Update node fields (content, confidence, tags, etc.)"""

    @abstractmethod
    def delete_node(self, node_id: str) -> bool:
        """Delete node and all its edges"""

    @abstractmethod
    def search_spreading(self, query: str, mode: str = "precise",
                         graph_dims: list = None, tags: list = None,
                         top: int = 5, layer: str = "L0") -> list:
        """SYNAPSE-style spreading activation search.
        mode: 'precise' (strong edges, high threshold)
              or 'creative' (weak edges, cross-dim, low threshold)"""

    @abstractmethod
    def match_preconditions(self, context_vector, top: int = 5) -> list:
        """Find memories whose precondition matches current context.
        Used for proactive injection."""

    @abstractmethod
    def verify_node(self, node_id: str) -> bool:
        """Mark node as verified: verified_count++, verified_at=now, confidence+=0.05"""

    # Existing methods unchanged
    def get_node(self, node_id) -> Optional[Dict]: ...
    def add_edge(self, from_id, to_id, relation_type, ...) -> str: ...
    def traverse(self, node_id, depth=2, ...) -> list: ...
    def search_by_vector(self, query, top=5, ...) -> list: ...  # kept for compat
    def search_by_keyword(self, query, top=5, ...) -> list: ...  # kept for compat
    def search_hybrid(self, query, top=5, ...) -> list: ...  # delegates to spreading
```

### 4.2 SQLiteStore Implementation Notes

- **FAISS Index**: `faiss.IndexFlatIP(1024)` built on startup from all node vectors
- **Precondition Index**: Separate `faiss.IndexFlatIP(1024)` for precondition_vec
- **Sync**: add_node → faiss.add(); delete_node → mark + periodic rebuild
- **graph_dim filtering**: SQL WHERE clause on edges, no schema change needed

## 5. Write Logic Upgrade (add_node)

### Current Flow
```
principle exact match → vector dedup (>0.92) → INSERT
```

### v6.0 Flow
```
1. principle exact match → reinforce old node (unchanged)
2. vector dedup (>0.92) → reinforce old node (unchanged)
3. NEW: encode precondition → precondition_vec (if provided)
4. NEW: set defaults:
   - half_life_days by type: experience=30, principle=90, strategy=60, correction=60, raw=15
   - context_tags = merge(task_type, project, user_tags)
5. INSERT node
6. NEW: predictive validation:
   a. IF precondition is set:
      - Search precondition_vec FAISS index for matches to current context
      - IF matched old memory has predicted_outcome:
        - Compare predicted_outcome with new memory's content
        - IF consistent → verify_node(old_id): verified_count++, confidence+0.05
        - IF contradictory → add_edge(new, old, "contradicts") + old.confidence -= 0.2
   b. IF no precondition:
      - Check if new principle contradicts existing principle (semantic check)
```

### Confidence Lifecycle

```
Write: confidence = 1.0
Verified (new similar experience confirms):  confidence += 0.05, max 1.5
Contradicted (new experience contradicts):   confidence -= 0.2, min 0.0
Dream decay: confidence *= e^(-ln(2) * days_since_verified / half_life)
Retrieval: final_score = similarity * decay_score * confidence
```

## 6. Dream Pipeline Restructuring

### 6.1 Fast Path (No LLM, Runs Every Time, ~5-10 sec)

| # | Phase | What Changed |
|---|-------|-------------|
| 1 | Snapshot | Unchanged |
| 2 | VectorIndex | **CHANGED**: Each new edge gets graph_dim + strength. New: precondition matching against existing predicted_outcomes |
| 3 | DecayRecalc | **CHANGED**: New formula with half_life and confidence |
| 4 | Covenant | Unchanged |
| 5 | Sync | Unchanged |

### 6.2 Slow Path (Uses LLM, Optional/Async, ~1-5 min)

| # | Phase | What Changed |
|---|-------|-------------|
| 6 | LogScan + Distill | Unchanged |
| 7 | Causal + CrossDomain | **CHANGED**: Added causal chain strategy distillation |
| 8 | ContradictionDetect | **CHANGED**: Semantic contradiction replaces keyword matching + A-MEM evolution writeback |
| 9 | StrategyGen | **CHANGED**: Distill from causal chains too |
| 10 | LLMReview | **CHANGED**: Only reviews Slow Path new edges + new contradictions |
| 11 | Audit | Unchanged |

### 6.3 Decay Formula Change

**Old**:
```
decay = base_score * e^(-0.03 * days) * log2(access_count + 2) * type_weight
```

**New**:
```
adjusted_half_life = half_life_days * (1 + log(verified_count + 1))
decay = base_score * e^(-ln(2) * days / adjusted_half_life)
        * log2(access_count + 2)
        * type_weight
        * confidence
```

Where:
- `half_life_days` = per-node field (default 30)
- `verified_count` = how many times confirmed by new experience
- `confidence` = 0-1 reliability score

### 6.4 Contradiction Detection Upgrade

**Old**: 8 keyword pairs ("失败" vs "成功", "错误" vs "正确", etc.)

**New**:
1. Vector similarity > 0.7 (same domain)
2. Content semantic opposition check: encode both contents, compute vector difference, check if difference magnitude exceeds threshold in the "opposition" direction
3. On detection:
   - Create `contradicts` edge (graph_dim=causal, strength=strong)
   - **A-MEM evolution**: Write back to old memory's abstract/overview: append "⚠️ May not apply after [date]"
   - Old memory: `confidence -= 0.2`

### 6.5 A-MEM Memory Evolution

When new memory links to old memory:
1. Check if old memory's `context_tags` should be updated with new tags
2. Check if old memory's `abstract` should incorporate new information
3. This is the core A-MEM insight: **memories are not append-only; old memories evolve**

## 7. Search Upgrade

### 7.1 SYNAPSE Spreading Activation

```python
def search_spreading(query, mode="precise", graph_dims=None, tags=None, top=5, layer="L0"):
    q_vec = embedder.encode(query)

    # Step 1: Seed nodes via FAISS
    seeds = faiss_index.search(q_vec, k=3)
    activation = {node_id: similarity for node_id, similarity in seeds}

    # Step 2: Spreading parameters
    if mode == "precise":
        decay_by_dim = {"semantic": 0.7, "causal": 0.8, "temporal": 0.6, "entity": 0.5}
        min_activation = 0.3
        max_hops = 2
        allowed_strength = ["strong"]
    else:  # creative
        decay_by_dim = {"semantic": 0.8, "causal": 0.9, "temporal": 0.7, "entity": 0.6}
        min_activation = 0.1
        max_hops = 3
        allowed_strength = ["strong", "weak"]

    # Step 3: Spread activation
    frontier = list(activation.keys())
    for hop in range(max_hops):
        next_frontier = []
        for node_id in frontier:
            edges = get_edges(node_id, graph_dims=graph_dims, strength=allowed_strength)
            for edge in edges:
                target = edge.other_node(node_id)
                new_activation = activation[node_id] * edge.weight * decay_by_dim[edge.graph_dim]
                if new_activation > min_activation:
                    if target not in activation or new_activation > activation.get(target, 0):
                        activation[target] = new_activation
                        next_frontier.append(target)
        frontier = next_frontier

    # Step 4: Lateral inhibition
    # For pairs of activated nodes connected by similar_to, suppress the weaker one
    for pair in get_similar_pairs(activation.keys()):
        weaker = min(pair, key=lambda x: activation[x])
        activation[weaker] *= 0.5

    # Step 5: Return top results with appropriate layer
    return sorted(activation.items(), key=lambda x: -x[1])[:top]
```

### 7.2 Dual-Channel Retrieval

| Channel | Use When | Edges | Threshold | Cross-Dim |
|---------|----------|-------|-----------|-----------|
| `precise` | Need reliable answer | strong only | 0.3 activation | No |
| `creative` | Brainstorming, "wild ideas" | strong + weak | 0.1 activation | Yes |

### 7.3 FAISS Integration

```python
class VectorIndex:
    def __init__(self, dimension=1024):
        import faiss
        self.index = faiss.IndexFlatIP(dimension)
        self.id_map = []  # faiss position → node_id

    def build_from_store(self, store):
        """Load all vectors from SQLite on startup"""
        nodes = store.bulk_get_vectors()
        if not nodes:
            return
        vectors = np.array([np.frombuffer(n["vector"], dtype=np.float32) for n in nodes])
        self.index.add(vectors)
        self.id_map = [n["id"] for n in nodes]

    def add(self, node_id, vector):
        self.index.add(vector.reshape(1, -1))
        self.id_map.append(node_id)

    def search(self, query_vec, top=5):
        scores, indices = self.index.search(query_vec.reshape(1, -1), top)
        return [(self.id_map[i], float(s)) for s, i in zip(scores[0], indices[0]) if i >= 0]
```

### 7.4 Precondition Matching (match_preconditions)

```python
def match_preconditions(self, context_vector, top=5):
    """Find memories whose precondition matches current context."""
    results = self.precondition_faiss.search(context_vector, top)

    output = []
    for node_id, similarity in results:
        node = self.get_node(node_id)
        if node and node.get("predicted_outcome"):
            output.append({
                "id": node_id,
                "precondition": node["precondition"],
                "predicted_outcome": node["predicted_outcome"],
                "confidence": node["confidence"],
                "similarity": similarity,
            })
    return output
```

## 10. Migration Script (migrate_v50_to_v60.py)

### Steps

1. **Backup**: `graph.db` → `graph.db.bak-v50`
2. **Add columns**:
   ```sql
   ALTER TABLE nodes ADD COLUMN confidence REAL DEFAULT 1.0;
   ALTER TABLE nodes ADD COLUMN verified_at TEXT;
   ALTER TABLE nodes ADD COLUMN verified_count INTEGER DEFAULT 0;
   ALTER TABLE nodes ADD COLUMN half_life_days REAL DEFAULT 30.0;
   ALTER TABLE nodes ADD COLUMN precondition TEXT;
   ALTER TABLE nodes ADD COLUMN predicted_outcome TEXT;
   ALTER TABLE nodes ADD COLUMN context_tags TEXT DEFAULT '[]';
   ALTER TABLE nodes ADD COLUMN precondition_vec BLOB;

   ALTER TABLE edges ADD COLUMN graph_dim TEXT DEFAULT 'semantic';
   ALTER TABLE edges ADD COLUMN strength TEXT DEFAULT 'strong';
   ```
3. **Backfill nodes**:
   ```sql
   UPDATE nodes SET confidence = MIN(1.0, decay_score / 0.8);
   UPDATE nodes SET verified_at = COALESCE(last_access, created_at);
   UPDATE nodes SET verified_count = MAX(1, access_count / 3);
   UPDATE nodes SET half_life_days = CASE type
     WHEN 'principle' THEN 90
     WHEN 'strategy' THEN 60
     WHEN 'correction' THEN 60
     WHEN 'raw' THEN 15
     ELSE 30 END;
   UPDATE nodes SET context_tags = json_group_array(value)
     FROM (SELECT id, value FROM nodes, json_each(
       '[' || COALESCE('"' || task_type || '"', 'null') || ',' ||
              COALESCE('"' || project || '"', 'null') || ']'
     ) WHERE value != 'null');
   ```
4. **Backfill edges**:
   ```sql
   UPDATE edges SET graph_dim = CASE relation_type
     WHEN 'similar_to' THEN 'semantic'
     WHEN 'is_a' THEN 'semantic'
     WHEN 'caused' THEN 'causal'
     WHEN 'solves' THEN 'causal'
     WHEN 'contradicts' THEN 'causal'
     WHEN 'transfers_to' THEN 'entity'
     WHEN 'evolved_from' THEN 'semantic'
     ELSE 'semantic' END;

   UPDATE edges SET strength = CASE
     WHEN weight >= 0.6 THEN 'strong'
     ELSE 'weak' END;
   ```
5. **Update meta**: `version` → `6.0.0`
6. **Verify**: count nodes/edges unchanged

## 14. Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| FAISS CPU dependency | Low | `pip install faiss-cpu`, pure Python, no system deps |
| Vector search perf at scale | Medium | FAISS O(log n) replaces O(n) full scan |
| Data loss during migration | High | Auto backup + rollback support |
| MCP backward compatibility | Medium | All new params optional, old calls unaffected |
| LLM cost for contradiction detection | Medium | Only runs in Slow Path, not on every write |
| Memory bloat from precondition_vec | Low | 4KB per node, 1000 nodes = 4MB |
| FAISS index drift after deletes | Low | Periodic rebuild from SQLite source of truth |
